What is Bash?

Bash is the shell.

A shell program is typically an executable binary that takes commands that you type and (once you hit return), translates those commands into system calls to the Operating System API. Through this, bash can read and execute commands from a file. This is called a shell script. Similar to Bash's predecisor and descendant Unix shells, it supports command substitutions, filename globbing (wildcard matching), here documents, piping, variables, and control structures for condition-testing and iteration.

Bash Scripting

One of the reasons Bash is so strong is that it can be scripted. Meaning that you can list everything from Bash in a plain-text file and have Bash run the commands for you. Compared to spending a whole day manually running a hundred commands, you can now script them and have your computer run them automatically while you finish other tasks. What makes bash even more amazing is that almost everything on Linux can run and scripted through Bash.

Basic Bash Commands

[ ]: Anything encased in [ ] means that it's optional. Some commands can be used without options or specifying files.

ls - List directory contents: Allows you to view all files within the specified directory

  • Syntax: ls [option(s)][file(s)]
  • Common options: -a, -l

echo - Prints text to the terminal window: typically used to show the output status text to the screen or computer file. It is also used for showing the values of environmental variables, which tell the shell how to behave as a user works at the command line or in scripts.

  • Syntax: echo [option(s)][string(s)]
  • Common options: -e, -n

touch - Creates a file: Used to create new files, change timestamps on files and/or directories. You can create as many files as you want in a single command without worrying about overwriting files with the same name.

  • Syntax: touch [option(s)][file_name(s)]
  • Common options: -a, -m, -r, -d

mkdir - Create a directory: Used to create new directories. Can be used to create multiple directories simultaneously.

  • Syntax: mkdir [option(s)][directory_name(s)]
  • Common options: -m, -p, -v

grep - search: used to search text for patterns specified by the user.

  • Syntax: grep [option(s)] pattern [file(s)]
  • Common options: -i, -c, -n

man - Print manual or get help for a command: Manual for when you forget what a certain command does.

  • Syntax: man [option(s)][keyword(s)]
  • Common options: -w, -f, -b

pwd - Print working directory: Used to print the current directory you're in.

  • Syntax: pwd [option(s)]
  • Common options: none

cd - Change directory: Used to change the directory you're in so that you can get info, manipulate, read, etc. the different files and directories in your system.

  • Syntax: cd [option(s)] directory
  • Common options: None

mv - Move or rename directory: Used to move or rename directories. Allows you to batch file renaming which saves you time.

  • Syntax: mv [option(s)] argument(s)
  • Common options: -i, -b

rm dir - Remove directory: Will remove empty directories. Helps clean and organize files and folders on the computer. It is important to note that there are two ways to remove directories: rm and rmdir. Rmdir will only delete empty directories while rm will remove directories and files regardless if they contain data or not.

  • Syntax: rmdir [option(s)] directory_names
  • Common options: -p

locate - Locate a specific file or directory: Simplest way to find a file or directory.

  • Syntax: locate [option(s)]file_name(s)
  • Common options: -q, -n, -i

less - view the contents of a text file: Allows you to view files without opening an editor. It is faster to use, and there's no chance of you inadvertently modifying the file.

  • Syntax: less file_name
  • Common options: -e, -f, -n

> - Redirect stdout: This takes the output from the preceding command that you'd normally see in the terminal and sends it to a file that you give it. As an example, take echo "contents of file1" > file1. Here it creates a file called file1 and puts the echoed string into it.

  • Syntax: >
  • Common options: N/a

cat - Read a file, create a file, and concatenate files: Serves three main functions: displaying them, combining copies of them, and creating new ones.

  • Syntax: cat [option(s)][file_name(s)][-][file_name(s)]
  • Common options: -n

| - Pipe: A pipe takes the standard output of one command and passes it as the input to another.

  • Syntax: |
  • Common options: N/a

head - Read the start of a file: Displays the first 10 lines of a file.

  • Syntax: head [option(s)][file(s)]
  • Common options: -n

tail - Read the end of a file: Displays the last 10 lines of a file.

  • Syntax: tail [option(s)] file_names
  • Common options: -n

chmod - Sets the file permissions flag on a file or folder: Used to change permissions on modifying a file that you don't have access to. You can use alphanumeric characters (u, g, o) and can be assigned their access with w, r, x. Conversely, you can also use octal number (0-7) to change the permissions. For example, chmod 777 my_file will give access to everyone.

  • Syntax: chmod [option(s)] permissions file_name
  • Common options: -f, -v

exit - Exit out of a directory: Will close a terminal indo, end the execution of a shell script or log you out of an SSH remote access session.

  • Syntax: exit
  • Common options: N/a

history - list your most recent commands: Lists our past commands you've used.

  • Syntax: history
  • Common options: -c, -d

clear - Clear your terminal window: Clears all previous commands and output from consoles and terminal indows. This keeps your terminal clean and removes the clutter.

  • Syntax: clear
  • Common options: -c, -d

cp - Copy files and directories: Use this command when you need to back up your files.

  • Syntax: cp [option(s)] current_name new_name
  • Common options: -r, -i, -b
