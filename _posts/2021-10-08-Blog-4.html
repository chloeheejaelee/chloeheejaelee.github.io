What is Docker?

Docker is an open-source project that is used for automating the deployment of apps that are portable, self-reliant containers that can run on the cloud or on-premises. Docker is mainly a containerization platform meaning it simply delivers and distributes applications, and has become really popular as organizations shift to cloud-native development and hybridge multi cloud environments. Although developers can create containers without Docker, the platform helps make the process faster, simpler, easier, and safer to build, deploy and manage containers. The isolation and security that is run through docker, allow many containers to be run simultaneously on a host. Lastly, these containers are easily shareable and allow multiple users to use the same container.

How Containers Work

Containers are built into the Linux kernel through the process of isolation and virtualization capabilities. There are two capabilities for Docker; Control group (C groups) is used for allocating resources among processes, while Namespaces is used for restricting a processes access or visibility into other resources or areas of the system. With these capabilities, it allows multiple app components to share the resources of a single instance of the host operating system. A good example is how a hypervisor allows multiple virtual machines (VMs) to share the CPU, memory and other resources of a single hardware server.
